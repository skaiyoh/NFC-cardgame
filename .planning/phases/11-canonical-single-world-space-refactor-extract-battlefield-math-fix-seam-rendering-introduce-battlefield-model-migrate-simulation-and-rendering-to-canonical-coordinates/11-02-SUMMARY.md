### Auto-fixed Issues

**1. [Rule 3 - Blocking] Resolved Vector2 typedef conflict between battlefield_math.h and Raylib**
- **Found during:** Task 1 (battlefield.h creation)
- **Issue:** battlefield_math.h defines its own Vector2 via VECTOR2_DEFINED guard, but Raylib also defines Vector2 without setting VECTOR2_DEFINED. Including battlefield_math.h before Raylib headers caused redefinition error.
- **Fix:** Reordered includes in battlefield.h to put tilemap_renderer.h (which brings Raylib) first, then explicitly define VECTOR2_DEFINED before including battlefield_math.h
- **Files modified:** src/core/battlefield.h
- **Verification:** make cardgame compiles cleanly
- **Committed in:** 1e16b95 (Task 1 commit)

**2. [Rule 3 - Blocking] Moved NUM_CARD_SLOTS and MAX_ENTITIES to config.h**
- **Found during:** Task 1 (battlefield.h creation)
- **Issue:** battlefield.h needs NUM_CARD_SLOTS and MAX_ENTITIES for array sizing but cannot include types.h (circular dependency). These constants were defined in types.h.
- **Fix:** Moved the #define statements to config.h (already included by battlefield.h). Removed duplicates from types.h (which includes config.h).
- **Files modified:** src/core/config.h, src/core/types.h
- **Verification:** make test && make cardgame both pass
- **Committed in:** 1e16b95 (Task 1 commit)

- **Fix:** Added src/core/battlefield_math.c to SRC_CORE in both Makefile and CMakeLists.txt
