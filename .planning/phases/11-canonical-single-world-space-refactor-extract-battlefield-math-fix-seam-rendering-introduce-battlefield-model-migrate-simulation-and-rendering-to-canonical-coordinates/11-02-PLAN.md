    1. `bf_side_for_player(int playerID)`: return `(playerID == 0) ? SIDE_BOTTOM : SIDE_TOP;`

    2. `bf_init(...)`:
       - Set `bf->boardWidth = BOARD_WIDTH`, `bf->boardHeight = BOARD_HEIGHT`, `bf->seamY = SEAM_Y`
       - Initialize bottom territory (SIDE_BOTTOM):
         - `bounds = {0, SEAM_Y, BOARD_WIDTH, BOARD_HEIGHT - SEAM_Y}` (per D-02: bottom = {0, 960, 1080, 960})

       - Initialize top territory (SIDE_TOP):

         - Copy biome data, create tilemap with `seedTop`
       - Generate canonical lane waypoints for both sides using a private helper `generate_canonical_waypoints(bf, side)`:
         - For SIDE_BOTTOM (P1): spawn at `depth=0.8 * territory.bounds.height` relative to territory bottom edge. Forward is toward decreasing y (per D-03).
           - slot spawn X: `territory.bounds.x + (slot + 0.5f) * laneWidth` where `laneWidth = territory.bounds.width / 3.0f`
           - slot spawn Y: `territory.bounds.y + territory.bounds.height * 0.8f`
           - Convert to canonical: these are already canonical because SIDE_BOTTOM identity maps
           - Waypoints march from spawnDepth=0.125 to endDepth=2.125 (same as pathfinding.c), using `player_lane_pos`-equivalent formula but in canonical coords:
             - For SIDE_BOTTOM: `y = territory.bounds.y + territory.bounds.height * (0.9 - depth * 0.8)` which yields motion toward y=0 (enemy end)
           - Apply `bow_offset` using same sinf formula as pathfinding.c (copy the static function)
         - For SIDE_TOP (P2): spawn at `depth=0.8 * territory.bounds.height` relative to territory. Forward is toward increasing y (per D-04).
           - Generate in side-local space first (same formula as SIDE_BOTTOM), then run through `bf_to_canonical` to apply lateral mirror (per D-06).
           - For Y: P2 forward is increasing y. In side-local space, use same formula but with direction inverted: `y = territory.bounds.y + territory.bounds.height * (0.1 + depth * 0.8)` so that depth=0 is near own base (top edge) and depth=2.125 goes past seam into opponent territory.
           - Actually simpler: generate using same depth formula as P1 but within a mirrored coordinate frame, then convert with bf_to_canonical for the lateral X flip only. The Y coordinates need to be generated to move in the correct direction for each side.
           - CONCRETE APPROACH: For each side, compute raw waypoints in side-local style, then wrap as SideLocalPos and convert via bf_to_canonical:
             - SIDE_BOTTOM local spawn: `(slotX, territoryY + 0.8*height)`, waypoints march toward `y=territory.y + height*(0.9 - depth*0.8)` which goes negative beyond territory
             - SIDE_TOP local spawn: for P2, local frame has own base at top of territory. Mirror the depth logic: `y = territory.bounds.y + territory.bounds.height * (0.1 + depth * 0.8)` which marches from near y=0 toward and past y=960
             - Lateral X in local space: `territory.bounds.x + (slot + 0.5f) * laneWidth` for both
             - Apply bow_offset to X in local space
             - Wrap as `SideLocalPos` with appropriate side, then `bf_to_canonical()` to get final canonical position
       - Store slot spawn anchors: `bf->slotSpawnAnchors[side][slot] = bf->laneWaypoints[side][slot_lane][0]` (first waypoint = spawn position, per existing convention where waypoint[0] == slot.worldPos)
       - Initialize entity registry: `bf->entityCount = 0`
