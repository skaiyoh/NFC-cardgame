Replace the entire `case ESTATE_WALKING:` block (lines 77-98) with waypoint-following logic. The new code:

1. **Validates lane and waypoint bounds** -- if `e->lane < 0 || e->lane >= 3`, break. If `e->waypointIndex >= LANE_WAYPOINT_COUNT`, transition to ESTATE_IDLE.

2. **Gets current target waypoint** from `owner->laneWaypoints[e->lane][e->waypointIndex]`.

3. **Moves toward waypoint** -- compute `diff = target - position`, `dist = length(diff)`, `step = moveSpeed * deltaTime`. If `dist <= step`, snap to target and increment waypointIndex. Otherwise, move by `step` in the normalized direction.

4. **End-of-path behavior** (per D-10, D-11) -- when `waypointIndex >= LANE_WAYPOINT_COUNT` after incrementing: apply random jitter within LANE_JITTER_RADIUS pixels on both X and Y axes, then call `entity_set_state(e, ESTATE_IDLE)`.

5. **Sprite direction from movement vector** (per D-07, D-08, D-09) -- compute dominant axis of `diff` vector. If `fabsf(diff.x) > fabsf(diff.y)`: set `e->anim.dir = DIR_SIDE` and `e->anim.flipH = (diff.x < 0)` (true = facing left). Else: set `e->anim.dir = (diff.y < 0) ? DIR_UP : DIR_DOWN` and `e->anim.flipH = false`.

Remove the old code entirely:
- Remove `e->position.y -= e->moveSpeed * deltaTime;` (straight-line movement)
- Remove `despawnY` calculation and `markedForRemoval` check (replaced by end-of-path idle)

Use manual vector math (sqrtf, direct component math) rather than raymath.h functions to avoid any include chain issues. The C standard `math.h` provides `sqrtf` and `fabsf`.

Concrete replacement code for the ESTATE_WALKING case:

```c
case ESTATE_WALKING: {
    Player *owner = &gs->players[e->ownerID];

    // Validate lane bounds
    if (e->lane < 0 || e->lane >= 3) break;

    // Check if already past all waypoints
    if (e->waypointIndex >= LANE_WAYPOINT_COUNT) {
        entity_set_state(e, ESTATE_IDLE);
        break;
    }

    Vector2 target = owner->laneWaypoints[e->lane][e->waypointIndex];
    Vector2 diff = { target.x - e->position.x, target.y - e->position.y };
    float dist = sqrtf(diff.x * diff.x + diff.y * diff.y);
    float step = e->moveSpeed * deltaTime;

    if (dist <= step || dist < 1.0f) {
        // Reached waypoint -- snap and advance
        e->position = target;
        e->waypointIndex++;

        if (e->waypointIndex >= LANE_WAYPOINT_COUNT) {
            // End of path: apply random jitter (per D-11)
            float jx = ((float)(rand() % ((int)(LANE_JITTER_RADIUS * 2) + 1)) - LANE_JITTER_RADIUS);
            float jy = ((float)(rand() % ((int)(LANE_JITTER_RADIUS * 2) + 1)) - LANE_JITTER_RADIUS);
            e->position.x += jx;
            e->position.y += jy;
            entity_set_state(e, ESTATE_IDLE);
        }
    } else {
        // Move toward waypoint
        float inv = 1.0f / dist;
        e->position.x += diff.x * inv * step;
        e->position.y += diff.y * inv * step;
    }

    // Update sprite direction from movement vector (per D-07, D-08, D-09)
    if (dist > 1.0f) {
        float ax = fabsf(diff.x);
        float ay = fabsf(diff.y);

        if (ax > ay) {
            // Dominant horizontal movement
            e->anim.dir = DIR_SIDE;
            e->anim.flipH = (diff.x < 0);  // true = facing left
        } else {
            // Dominant vertical movement
            e->anim.dir = (diff.y < 0) ? DIR_UP : DIR_DOWN;
            e->anim.flipH = false;
        }
    }

    break;
}
```

    for (int lane = 0; lane < 3; lane++) {

            // Draw waypoint dot

                DrawLineV(p->laneWaypoints[lane][i],

            }
        }
    }
}
