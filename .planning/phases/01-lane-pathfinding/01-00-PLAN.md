   - Config defines: `LANE_WAYPOINT_COUNT 8`, `LANE_BOW_INTENSITY 0.5f`, `LANE_JITTER_RADIUS 10.0f`, `PI_F 3.14159265f`
   - Enums: `EntityState` with `ESTATE_IDLE, ESTATE_WALKING, ESTATE_DEAD`
   - Enums: `SpriteDirection` with `DIR_DOWN, DIR_SIDE, DIR_UP`
   - Enums: `AnimationType` with at minimum `ANIM_IDLE, ANIM_WALK`
   - Minimal `AnimState` struct: `{ AnimationType anim; SpriteDirection dir; int frame; float timer; float fps; bool flipH; }`
   - Minimal `CardSlot` stub: `{ Vector2 worldPos; void *activeCard; float cooldownTimer; }`
   - `NUM_CARD_SLOTS 3`, `MAX_ENTITIES 64`
   - Minimal `Entity` struct with: `int id; int type; int faction; EntityState state; Vector2 position; float moveSpeed; int hp, maxHP; int attack; float attackSpeed; float attackRange; AnimState anim; void *sprite; float spriteScale; int ownerID; int lane; int waypointIndex; bool alive; bool markedForRemoval;`
   - Minimal `Player` struct with: `int id; Rectangle playArea; Rectangle screenArea; /* camera stub */ CardSlot slots[3]; void *entities[64]; int entityCount; float energy; float maxEnergy; float energyRegenRate; void *base; Vector2 laneWaypoints[3][LANE_WAYPOINT_COUNT];`
   - Note: Camera2D, TileMap, BiomeDef, etc. can be omitted or replaced with padding bytes since pathfinding.c only accesses `playArea`, `slots`, and `laneWaypoints`. However, the struct layout must match for `laneWaypoints` to be at the correct offset. The simplest approach: define the Player struct with the exact same field order as types.h but replace complex types with `char padding[N]` of equivalent size. Actually, since we include pathfinding.c directly and it only accesses `playArea`, `slots[lane].worldPos`, and `laneWaypoints`, AND we control the struct definition, we can use a simplified Player struct where laneWaypoints directly follows the fields pathfinding.c uses.

   **CRITICAL:** Since we `#include "pathfinding.c"` directly, pathfinding.c has `#include "pathfinding.h"` which has `#include "../core/types.h"`. We must prevent that. Instead, use this approach:
   - Define all stubs and types FIRST
   - Define guard macros to prevent pathfinding.h and types.h from being included: `#define NFC_CARDGAME_PATHFINDING_H` and `#define NFC_CARDGAME_TYPES_H`
   - Also define `#define NFC_CARDGAME_PLAYER_H` to block player.h
   - Then `#include "../src/logic/pathfinding.c"` -- since the header guards are already defined, the `#include` directives inside pathfinding.c will be no-ops
